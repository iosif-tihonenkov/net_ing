### Инструкция по настройке IPv4‑адресации для удалённого доступа к устройствам Cisco

#### 1. Основные понятия

* **SVI (Switched Virtual Interface)** — виртуальный интерфейс коммутатора (часто называется интерфейсом VLAN), на который назначается IP‑адрес управления.
* **VLAN** — виртуальная локальная сеть; по умолчанию все порты коммутатора Cisco находятся в VLAN 1.
* **Шлюз по умолчанию** (`default gateway`) — IP‑адрес маршрутизатора, через который коммутатор отправляет пакеты за пределы локальной подсети.

#### 2. Настройка статического IP‑адреса


**Пошаговая инструкция:**


1. Войдите в режим глобальной конфигурации:
   ```
   enable
   configure terminal
   ```
2. Перейдите в режим конфигурации VLAN‑интерфейса (например, VLAN 1):
   ```
   interface vlan 1
   ```
3. Назначьте IP‑адрес и маску подсети:
   ```
   ip address 192.168.1.200 255.255.255.0
   ```
4. Включите интерфейс (если он отключён):
   ```
   no shutdown
   ```
5. Настройте шлюз по умолчанию:
   ```
   ip default-gateway 192.168.1.1
   ```
6. (Опционально) Настройте DNS‑серверы:
   ```
   ip name-server 8.8.8.8 8.8.4.4
   ```

**Полный пример настройки:**
```
enable
configure terminal
interface vlan 1
ip address 192.168.1.200 255.255.255.0
no shutdown
ip default-gateway 192.168.1.1
ip name-server 8.8.8.8 8.8.4.4
```

#### 3. Настройка получения IP‑адреса по DHCP


**Пошаговая инструкция:**

1. Войдите в режим конфигурации VLAN‑интерфейса:
   ```
   enable
   configure terminal
   interface vlan 1
   ```
2. Включите интерфейс:
   ```
   no shutdown
   ```
3. Настройте получение IP‑адреса через DHCP:
   ```
   ip address dhcp
   ```

**Полный пример настройки:**
```
enable
configure terminal
interface vlan 1
no shutdown
ip address dhcp
```

#### 4. Проверка настроек

Используйте следующие команды для проверки конфигурации:

* **`show running-config`** — отображает текущую конфигурацию, включая IP‑адресацию.
* **`show interfaces vlan <номер>`** — показывает состояние и IP‑адрес указанного VLAN‑интерфейса. Обращайте внимание на:
    * состояние интерфейса (`up/up` — работает, `administratively down` — отключён);
    * IP‑адрес интерфейса.
* **`show dhcp lease`** — отображает арендованный IP‑адрес и параметры DHCP (при использовании DHCP).
* **`ping <IP‑адрес>`** — проверяет связность с указанным узлом.

**Примеры вывода команд:**

* Вывод `show interfaces vlan 1` (фрагмент):
  ```
  Vlan1 is up, line protocol is up
  Internet address is 192.168.1.200/24
  ```
* Вывод `show dhcp lease` (фрагмент):
  ```
  IP address: 192.168.1.50
  Subnet mask: 255.255.255.0
  Gateway: 192.168.1.1
  DNS server: 8.8.8.8
  Lease expiration: 2024/04/30 10:00:00
  ```

---

#### Важные замечания и рекомендации


* **Состояние интерфейса.** Без выполнения команды `no shutdown` интерфейс останется в состоянии `administratively down`, и коммутатор не сможет обмениваться пакетами.
* **Привязка VLAN к физическому порту.** VLAN‑интерфейс должен быть привязан к хотя бы одному активному физическому порту коммутатора. Иначе интерфейс останется в состоянии `down`.
* **Выбор VLAN.** Для управления можно использовать любой VLAN (не обязательно VLAN 1). Рекомендуется выделять отдельный VLAN для управления сетью — это повышает безопасность.
* **Сохранение конфигурации.** Настройки статического IP сохраняются в `running-config`. Настройки, полученные по DHCP, не сохраняются в конфигурации — они теряются после перезагрузки.
* **Шлюз по умолчанию.** Необходим для связи с устройствами за пределами локальной подсети (например, для удалённого управления через Telnet/SSH или работы SNMP).
* **DNS‑серверы.** Опциональная настройка. Упрощает удалённое управление, позволяя использовать имена хостов вместо IP‑адресов.
