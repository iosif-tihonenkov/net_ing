# Что такое ARP? 

ARP - Address Resolution Protocol- Протокол, который служит для сопоставления MAC- и IP-адресов. На канальном уровне - MAC, на сетевом - собственно IP. 

Адреса 2 уровня OSI - используются для локальной передачи между напрямую подключенными устройствами. Адреса 3-го уровня - для устройств в разных сетях.

Протокол Ethernet использует MAC-адреса, привязанные к сетевой карте.

Для связи устройств из разных сетей MAC-адрес должен быть сопоставлен с IP-адресом. Для этого используются следующие протоколы:

1. Address Resolution Protocol (ARP)
2. Reverse ARP (RARP) - 
3. Serial Line ARP (SLARP)
4. Inverse ARP (InARP) - по известному MAC-адресу выдается IP
5. Proxy ARP (PARP) - для общения внутри одной сети, но разделенной маршрутизатором (разные LAN)

Так как постоянный запрос в сеть ARP будет снижать производительность сети, то для минимизации широковещательных запросов реализовано кэширование ARP.
ARP-кэш хранит в виде таблице соответствие IP- и MAC-адресов в памяти на определенное время. То есть переодический опрос сети все еще присутствует, но уже хотя бы не постоянный и не грузит так сеть. 
Ну и получается, что т.к. на один запрос/ответ меньше, то и проходить информация будет быстрее.

Записи в ARP-таблице бывают статические и динамические. 

Динамические - понятно, получаются в результате опроса сети, а статические - в результате ручного добавления. Однако и поддержка/обновление информации в этой таблице - либо динамическое, либо ручное. Кому как удобнее.
