## Базовая настройка коммутатора Cisco

### Режимы работы сетевых устройств

Режимы работы делятся на три категории:

1. **Плоскость передачи данных (data plane)** — основная функция коммутатора: приём и пересылка кадров от подключённых устройств.
2. **Плоскость управления (control plane)** — определяет работу data plane: состояние интерфейсов, скорость портов, блокировка портов через spanning tree и т. д.
3. **Плоскость наблюдения (management plane)** — управление самим устройством.

В этом уроке рассматриваются базовые функции управления коммутатором Cisco: настройка безопасности входа, конфигурирование IPv4‑адресации для удалённого доступа и вспомогательные настройки для администратора.

---

### Защита коммутатора через CLI

По умолчанию коммутатор Cisco Catalyst:
* не требует пароля для подключения к консольному порту;
* даёт полный доступ к настройкам при физическом подключении;
* полностью закрывает доступ по Telnet и SSH.

**Цель настройки безопасности:** разрешить доступ к CLI только авторизованным пользователям.

### Настройка парольного доступа к коммутатору Cisco

#### 1. Защита пользовательского и привилегированного режимов с помощью простых паролей

**Шаг 1.** Задайте пароль на привилегированный режим:
```
enable secret <пароль>
```
* Предпочтительно использовать `enable secret` вместо `enable password` — он обеспечивает более высокий уровень безопасности.

**Шаг 2.** Задайте пароль на доступ по консоли:
```
line con 0
password <пароль>
login
```

**Шаг 3.** Задайте пароль на терминальные подключения (vty, Telnet):
```
line vty 0 15
password <пароль>
login
```

**Пример полной настройки:**
```
Switch# configure terminal
Switch(config)# enable secret cisco
Switch(config)# line console 0
Switch(config-line)# password cisco
Switch(config-line)# login
Switch(config-line)# exit
Switch(config)# line vty 0 15
Switch(config-line)# password cisco
Switch(config-line)# login
Switch(config-line)# end
Switch#
```

#### 2. Защита доступа в пользовательском режиме с использованием локальной базы данных

**Преимущества:** использование пар «имя пользователя / пароль» вместо общего пароля.

**Шаги настройки:**

**Шаг 1.** Создайте пары «имя пользователя / пароль» в локальной базе:
```
username <имя пользователя> secret <пароль>
```
**Пример:**
```
username ulanbaby secret box
username landy secret man
```

**Шаг 2.** Настройте консоль на использование локальной базы:
```
line con 0
login local
no password
```

**Шаг 3.** Настройте Telnet (vty) на использование локальной базы:
```
line vty 0 15
login local
no password
```

#### 3. Защита доступа в пользовательском режиме с помощью внешних серверов аутентификации (AAA)

**Преимущества AAA‑сервера:**
* централизованное хранилище учётных записей;
* принудительная периодическая смена паролей;
* блокировка учётных записей уволенных сотрудников и т. д.

**Принцип работы:**
1. Пользователь вводит имя и пароль.
2. Коммутатор отправляет запрос на AAA‑сервер (по протоколам RADIUS или TACACS+).
3. Сервер подтверждает подлинность.
4. Коммутатор разрешает доступ.

#### 4. Защита удалённого доступа с помощью Secure Shell (SSH)

**Преимущества SSH перед Telnet:**
* шифрование всего трафика (включая пароли);
* защита данных от перехвата.

**Пошаговая настройка SSH:**

**Шаг 1.** Настройте генерацию пары открытого и закрытого ключей:
1. Задайте имя коммутатора:
   ```
   hostname <имя>
   ```
2. Задайте доменное имя:
   ```
   ip domain-name <домен>
   ```
3. Сгенерируйте RSA‑ключ (рекомендуемый размер — не менее 768 бит для SSHv2):
   ```
   crypto key generate rsa
   ```
   Или с указанием размера ключа:
   ```
   crypto key generate rsa modulus <размер>
   ```

**Шаг 2** (опционально). Ограничьте доступ только SSHv2:
```
ip ssh version 2
```

**Шаг 3** (опционально). Настройте протоколы подключения на линиях vty:
* разрешить только SSH:
  ```
  transport input ssh
  ```
* разрешить SSH и Telnet:
  ```
  transport input telnet ssh
  ```
* разрешить все протоколы (по умолчанию):
  ```
  transport input all
  ```

**Шаг 4.** Настройте локальную аутентификацию на линиях vty (если используется):
```
line vty 0 15
login local
```

### Команды для проверки состояния SSH

* `show ip ssh` — информация о состоянии SSH‑сервера.
* `show ssh` — информация об активных SSH‑клиентах, подключённых к коммутатору.

**Пример вывода:**
```
Switch# show ip ssh
SSH Enabled - version 2.0...
Switch# show ssh
% No SSH session is active
```
