# Port‑Security на коммутаторах Cisco: настройка и использование

## Что такое Port‑Security

**Port‑Security** — функция коммутатора, которая контролирует, каким устройствам разрешено передавать трафик через определённый порт. Идентификация устройств происходит по их **MAC‑адресам**.

Основные задачи функции:
* защита от несанкционированных подключений;
* предотвращение атак на переполнение таблицы MAC‑адресов.

Способы настройки доступа:
* явное указание разрешённых MAC‑адресов;
* задание максимального количества MAC‑адресов, которые могут передавать трафик через порт.

## Типы Port‑Security

### 1. Статические MAC‑адреса

MAC‑адреса прописываются вручную на порту из режима конфигурации интерфейса:

```
switchport port-security mac-address [MAC-адрес]
```

Особенности:
* адреса сохраняются в таблице адресов;
* добавляются в текущую конфигурацию коммутатора.

### 2. Динамические MAC‑адреса

Коммутатор автоматически изучает MAC‑адреса и хранит их только в таблице адресов.

Особенности:
* при перезапуске коммутатора адреса удаляются;
* не сохраняются в конфигурации.

### 3. Sticky MAC‑адреса

MAC‑адреса, которые:
* изучены динамически;
* заданы вручную;
* сохраняются в таблице адресов и добавляются в текущую конфигурацию.

#### Настройка Sticky‑обучения

Чтобы динамически изученные адреса автоматически попадали в текущую конфигурацию, включается **sticky‑обучение**:

```
switchport port-security mac-address sticky
```

Что происходит после ввода команды:
* коммутатор преобразует все динамически изученные MAC‑адреса (в т. ч. изученные до включения sticky‑обучения) в sticky‑адреса;
* все адреса добавляются в таблицу адресов и текущую конфигурацию.

#### Ручное задание Sticky‑адресов

Можно задать адреса вручную:

```
switchport port-security mac-address sticky [MAC-адрес]
```

Особенности:
* указанные адреса сразу добавляются в таблицу адресов и текущую конфигурацию;
* если адреса сохранены в файле конфигурации, после перезапуска коммутатора или отключения интерфейса переобучение не потребуется;
* при отключении sticky‑обучения командой `no switchport port-security mac-address sticky` адреса остаются в таблице, но удаляются из текущей конфигурации.

## Нарушение безопасности и режимы реагирования

Port Security фиксирует нарушение в двух случаях:
1. На интерфейсе достигнут лимит MAC‑адресов, и устройство с неизвестным MAC пытается получить доступ к порту.
2. MAC‑адрес, уже закреплённый за одним интерфейсом, появляется на другом интерфейсе в той же VLAN.

### Режимы реагирования на нарушение

| Режим | Передача трафика | Отправка syslog | Сообщение об ошибке | Счётчик нарушений | Выключение порта |
|------|---------------|--------------|------------------|-----------------|----------------|
| **Protect** | Нет | Нет | Нет | Нет | Нет |
| **Restrict** | Нет | Да | Нет | Да | Нет |
| **Shutdown** (по умолчанию) | Нет | Нет | Нет | Да | Да |

**Описание режимов:**

* **Protect**: при достижении лимита MAC‑адресов пакеты с неизвестными источниками отбрасываются, пока не освободится место или не увеличится лимит. Уведомления не отправляются.
* **Restrict**: работает как Protect (лишние пакеты отбрасываются), но дополнительно:
    * отправляется уведомление о нарушении;
    * увеличивается счётчик ошибок.
* **Shutdown**: режим по умолчанию. При нарушении:
    * порт немедленно переходит в состояние **error‑disabled**;
    * гаснет светодиод;
    * увеличивается счётчик нарушений.

Для восстановления порта после срабатывания Shutdown нужно ввести команды:
```
shutdown
no shutdown
```
в режиме конфигурации интерфейса.

Изменение режима нарушения:
```
port-security violation {protect \| restrict \| shutdown}
```

## Настройка Port‑Security: практический пример

Пошаговая инструкция по настройке:

1. Заходим в режим конфигурации порта:
```
Switch# interface fa0/1
```
2. Устанавливаем порт в режим access:
```
Switch(config-if)# switchport mode access
```
3. Включаем Port‑Security:
```
Switch(config-if)# switchport port-security
```
4. Задаём максимальное количество адресов на порту (в примере — 50):
```
Switch(config-if)# switchport port-security maximum 50
```
5. Включаем sticky‑обучение:
```
Switch(config-if)# switchport port-security mac-address sticky
```

**Важно:** если включить Port‑Security командой `switchport port-security` без дополнительных параметров, то по умолчанию:
* максимальное количество адресов на порту — **1**;
* sticky‑обучение — **выключено**;
* режим нарушения — **shutdown**.

## Проверка настроек Port‑Security

Для проверки параметров и состояния Port‑Security используются следующие команды:

* **Просмотр параметров Port‑Security для конкретного интерфейса:**
```
show port-security [номер_интерфейса]
```
Пример:
```
show port-security fa0/1
```
* **Просмотр всех защищённых MAC‑адресов:**
```
show port-security address
```

## Краткий итог

Port‑Security — мощный инструмент для повышения сетевой безопасности на уровне коммутатора. Он позволяет:
* контролировать доступ устройств по MAC‑адресам;
* ограничивать количество подключений на порту;
* гибко настраивать реакцию на нарушения безопасности.

Правильная настройка Port‑Security помогает предотвратить:
* несанкционированный доступ к сети;
* атаки на переполнение таблицы MAC‑адресов;
* конфликты из‑за дублирования MAC‑адресов в VLAN.
