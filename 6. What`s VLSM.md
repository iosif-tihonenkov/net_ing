# Что такое VLSM - разбиение сети на подсети

Существует две версии IP - 4 и 6 версия. 

V4: 32-битное двоичное число поделенное на 4 октета ввиде десятеричных чисел. Пример: 192.168.0.1
V6: 128-битное. Все практически аналогично, только октетов 8. В каждом октете получается 16 бит, опять же - в двоичном виде. Для удобства - пишут в шестнадцеричном виде, разделяя двоеточиями. Пример: 2003:00af:cafe:3daf:1000:edaf:1001:afad

IP делятся на публичные и приватные. Публичные - белые, видны в интернете, приватные - внутренние, видны только внутри локалки. 

Для приватных зарезервированы три диапазона: 
1. 10.0.0.0 - 10.255.255.255
2. 172.16.0.0 - 172.31.255.255
3. 192.168.0.0 - 192.168.255.255

Адрес 0.0.0.0 зарезервирован для хоста, который только подключился к сети и еще не получил IP. 
Адрес 255.255.255.255 широковещательный адрес
Адрес 127.х.х.х - зарезервирован для loopback-адресации

IP адрес состоит из двух частей - адрес сети и адрес узла
Адрес сети считается по количеству единиц в левой части адреса - маска. 

К примеру: 
192.168.25.0/24 - подразумевается, что маска здесь - из 24 единиц (остальное нули). 

Если переписывать маску в бинарную систему - 1111 1111.1111 1111.1111 1111.0000 0000. Здесь 24 единицы. Получается, что у нас остается только крайний октет. Это числа с 0 до 255. 0 и 255 выкидываем. 
Получается диапазон - 192.168.25.1 - 192.168.25.254
